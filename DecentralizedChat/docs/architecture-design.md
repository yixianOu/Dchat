#### 3.1 Hub集群高可用
- 3+节点组成NATS Routes全互联集群，JetStream流配置3副本冗余，单节点故障无数据丢失
- **动态扩容**：支持在线添加新节点，零停机，Raft自动同步数据，业务完全无感知
- **容错能力**：3节点集群可容忍1台节点故障，5节点可容忍2台故障

#### 3.2 终端高可用

## 架构验证结论（2026-03-15 更新）

### 一、JetStream集群动态扩容技术验证结论
经过多轮测试验证，NATS JetStream集群存在**硬性设计限制**：
> ❌ **JetStream集群不支持无配置自动节点发现**
>
> 错误信息：`JetStream cluster requires configured routes or solicited leafnode for the system account`
>
> 原因：JetStream基于Raft共识算法实现强一致性，要求集群成员列表必须预先明确配置，不允许未知节点自动加入集群，否则会破坏Raft共识的一致性保证。

### 二、集群模式特性对比
| 集群模式 | 自动发现 | 动态扩容 | JetStream支持 | 去中心化程度 | 适用场景 |
|---------|---------|---------|--------------|-------------|---------|
| **纯Routes集群** | ✅ 支持链式自动发现 | ✅ 零停机动态添加新节点 | ❌ 不支持持久化 | ✅ 完全去中心化 | P2P消息路由、终端节点直连 |
| **JetStream集群** | ❌ 不支持自动发现 | ⚠️ 需要预配置或Operator管理 | ✅ 支持流持久化 | ⚠️ 半中心化 | Hub层持久化、离线消息存储 |
| **LeafNode模式** | ✅ LeafNode自动发现Hub | ✅ 终端动态接入 | ✅ Hub层支持持久化 | ✅ 终端去中心化+Hub高可用 | ✅ **生产级推荐架构** |

### 三、动态扩容方案建议
#### 1. 生产级最优方案（LeafNode架构）
```
                    ┌─────────────────────────────────┐
                    │   Hub Cluster (Routes全互联)    │
                    │  ┌──────┐  ┌──────┐  ┌──────┐  │
用户终端LeafNode ←───→│  │ Hub1 │  │ Hub2 │  │ Hub3 │  │ ←─── 用户终端LeafNode
                    │  └──────┘  └──────┘  └──────┘  │
                    │  ✅ Routes全互联  ✅ JetStream 3副本  │
                    └─────────────────────────────────┘
```
**架构说明**：
- Hub层：3+节点组成**全互联的Routes集群**，消息可以在所有Hub节点之间自动路由
- JetStream层：3个Hub节点预先配置为JetStream Raft共识组成员，流数据3副本冗余
- 终端层：用户设备运行LeafNode，配置多个Hub地址，自动选择可用Hub接入

**核心优势**：
- ✅ 终端用户无需预配置，只需配置多个Hub地址即可自动接入
- ✅ LeafNode支持多Hub自动健康检查，故障秒切到其他可用Hub
- ✅ 不管LeafNode连接哪个Hub，消息都可以通过Hub Routes集群路由到所有用户
- ✅ JetStream流是全局的，从任意Hub都能读到完整的消息，3副本冗余保证单节点故障无数据丢失
- ✅ 符合官方推荐的"边缘计算+中心持久化"分层架构

#### 2. 纯Hub集群动态扩容方案
如果需要扩展Hub JetStream集群，有两种实现方式：
- **方案A（预配置模式）**：在所有Hub节点的配置文件中预先写入所有可能加入集群的节点地址，新节点启动后会自动被Raft集群识别
- **方案B（Operator模式）**：在K8s环境中使用官方NATS Operator管理集群，支持一键扩缩容，自动调整Raft成员列表

### 四、现有方案修正
原方案中"Hub集群支持在线动态添加新节点，零停机，Raft自动同步数据"的描述需要修正：
> ✅ **正确描述**：
> - 3+节点组成预配置的JetStream集群，流配置3副本冗余，单节点故障无数据丢失
> - 扩容新Hub节点需要预先将新节点地址配置到所有现有节点的路由表中
> - 配置完成后新节点可零停机加入，Raft自动同步数据，业务无感知
