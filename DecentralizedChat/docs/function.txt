  项目代码架构设计与逻辑功能分析

  一、整体架构概览

  DecentralizedChat 是一个基于 NATS Routes 集群 + Wails 框架
  的企业级去中心化聊天应用。采用混合架构设计：

  ┌─────────────────────────────────────────────────────────────┐
  │                   React 前端 (TypeScript)                    │
  │  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐ │
  │  │ App.tsx  │  │ChatRoom  │  │KeyManager│  │  Services│ │
  │  └──────────┘  └──────────┘  └──────────┘  └──────────┘ │
  └─────────────────────────────────────────────────────────────┘
                              ↕ Wails 绑定
  ┌─────────────────────────────────────────────────────────────┐
  │                    Go 后端 (app.go)                          │
  │  ┌──────────────────────────────────────────────────────┐  │
  │  │  App 主应用 (统一编排)                               │  │
  │  └──────────────────────────────────────────────────────┘  │
  └─────────────────────────────────────────────────────────────┘
          ┌──────────────┬──────────────┬──────────────┐
          ↓              ↓              ↓              ↓
  ┌──────────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
  │ internal/chat│ │internal/ │ │internal/ │ │internal/ │
  │   (聊天服务)  │ │  nats    │ │  config  │ │  routes  │
  └──────────────┘ └──────────┘ └──────────┘ └──────────┘
          ↓              ↓              ↓              ↓
  ┌─────────────────────────────────────────────────────────────┐
  │              NATS Routes 集群 (去中心化网络)                  │
  │  ┌──────────┐    ┌──────────┐    ┌──────────┐            │
  │  │Public-1  │◄──►│Public-2  │◄──►│Public-3  │            │

  1. App 主应用层 (app.go)

  职责：统一编排所有内部模块，提供 Wails 前端绑定接口

  核心逻辑流程：
  OnStartup 启动流程：
    │
    ├─> 1. 加载配置 (config.LoadConfig)
    │     └─> 从 ~/.dchat/config.json 读取
    │
    ├─> 2. 初始化简化 NSC 设置 (nscsetup.EnsureSimpleSetup)
    │     └─> 生成 Operator/Account/User 密钥
    │
    ├─> 3. 创建 NodeManager (routes.NewNodeManager)
    │     └─> 用于管理嵌入式 NATS Server
    │
    ├─> 4. 启动 NATS 节点 (startNATSNode)
    │     └─> 启动嵌入式 NATS Server + Routes 集群
    │
    ├─> 5. 创建 NATS 客户端服务 (nats.NewService)
    │     └─> 连接到本地 NATS Server
    │
    ├─> 6. 创建聊天服务 (chat.NewService)
    │     └─> 初始化加密、密钥管理、消息处理
    │
    ├─> 7. 注册事件回调
    │     ├─> OnDecrypted → 推送到前端 "message:decrypted"
    │     └─> OnError → 推送到前端 "message:error"
    │
    └─> 8. 加载 NSC 密钥用于聊天加密
          └─> 从 .seed 文件读取，派生聊天密钥对

  2. 聊天服务模块 (internal/chat/service.go)

  职责：端到端加密通信、密钥管理、消息收发

  核心数据结构：
  type Service struct {
      nats          *natsservice.Service
      user          *User
      userPrivB64   string      // 本地私钥 (Base64)
      userPubB64    string      // 本地公钥 (Base64)
      nscKeyManager *NSCKeyManager  // ⭐ NSC 密钥管理器

      // 密钥缓存 (内存 + KV 二层存储)
      friendPubKeys map[string]string  // uid -> pubkey
      groupSymKeys  map[string]string  // gid -> symkey

      // 活跃订阅
      directSubs    map[string]*nats.Subscription
      groupSubs     map[string]*nats.Subscription

      // 事件处理器
      handlers      []func(*DecryptedMessage)
      errHandlers   []func(error)
  }

  核心逻辑流程：

  A. 消息发送流程 (SendDirect)
  SendDirect(peerID, content)
    │
    ├─> 1. 获取本地私钥
    │
    ├─> 2. 按需获取好友公钥 (getFriendKey)
    │     ├─> 先查内存缓存
    │     └─> 缓存未命中 → 从 NATS KV 查询
    │
    ├─> 3. 派生会话 ID (deriveCID)
    │     └─> SHA256(min(uid1,uid2) + ":" + max(uid1,uid2))[:16]
    │
    ├─> 4. 加密消息 (encryptDirect)
    │     ├─> X25519 密钥交换
    │     ├─> XSalsa20-Poly1305 加密
    │     └─> 24字节随机 nonce
    │
    ├─> 5. 构造加密消息 (encWire)
    │     {cid, sender, ts, nonce, cipher}
    │
    └─> 6. 发布到 NATS 主题
          └─> "dchat.dm.{cid}.msg"

  B. 消息接收流程 (handleEncrypted)
  NATS 订阅回调收到消息
    │
    ├─> 1. 反序列化 encWire
    │
    ├─> 2. 忽略本地自发回环
    │
    ├─> 3. 判断消息类型 (私聊/群聊)
    │
    ├─> 4. 按需获取解密密钥
    │     ├─> 私聊: getFriendKey(sender)
    │     └─> 群聊: getGroupKey(cid)
    │
    ├─> 5. 解密消息
    │     ├─> 私聊: decryptDirect (NaCl Box)
    │     └─> 群聊: decryptGroup (AES-256-GCM)
    │
    ├─> 6. 构造 DecryptedMessage
    │
    └─> 7. 触发 OnDecrypted 事件回调
          └─> 推送到前端

  C. 密钥懒加载机制 ⭐
  getFriendKey(uid):
    │
    ├─> 1. 检查内存缓存 friendPubKeys[uid]
    │     └─> 命中 → 立即返回
    │
    ├─> 2. 缓存未命中 → 从 NATS KV 查询
    │     └─> nats.GetFriendPubKey(uid)
    │
    └─> 3. 查询成功 → 缓存到内存 → 返回

  优点：
  - 启动时不需要预加载所有密钥
  - 内存占用低
  - 按需查询，性能优化

  三、加密与安全体系

  1. NSC 密钥派生 (internal/chat/nsc_crypto.go)

  设计思想：从 NSC 种子密钥 (SU...) 派生出专门用于聊天的密钥对

  派生流程：
  NSC User Seed (SU...)
    │
    ├─> 1. 解析为 Ed25519 私钥
    │
    ├─> 2. 计算域分隔键
    │     └─> HMAC-SHA256(key=seed, data="dchat-chat-v1")
    │
    ├─> 3. 派生聊天 Ed25519 密钥对
    │
    └─> 4. 转换为 X25519 用于加密
          (Ed25519 和 X25519 密钥可互换)

  优点：
  - 用户只需备份一个 NSC seed
  - 不同用途使用不同密钥 (隔离风险)
  - 与 NATS NSC 认证系统统一

  2. 端到端加密算法

  ┌──────────┬───────────────────────────────────────┬────────────────────┐
  │   场景   │                 算法                  │        说明        │
  ├──────────┼───────────────────────────────────────┼────────────────────┤
  │ 私聊加密 │ NaCl Box (X25519 + XSalsa20-Poly1305) │ 公钥加密，前向安全 │
  ├──────────┼───────────────────────────────────────┼────────────────────┤
  │ 群聊加密 │ AES-256-GCM                           │ 对称加密，高性能   │
  ├──────────┼───────────────────────────────────────┼────────────────────┤
  │ 身份签名 │ Ed25519 (NSC)                         │ 数字签名验证身份   │
  └──────────┴───────────────────────────────────────┴────────────────────┘

  四、网络架构

  1. NATS 主题设计

  私聊消息: dchat.dm.{cid}.msg
    └─ cid = SHA256(uid1, uid2)[:16]

  群聊消息: dchat.grp.{gid}.msg
    └─ gid = 群组标识符

  临时收件箱: _INBOX.{unique-id}
    └─ 用于请求-响应模式

  2. Routes 集群发现

  初始状态：
    NodeA (种子节点)

  添加 NodeB：
    NodeB ──连接──> NodeA
    NodeB <──获取所有节点地址── NodeA
    NodeB ──自动连接──> NodeA, NodeC, NodeD...

  最终形成全网状网络：
    NodeA ◄──► NodeB
      │  \    /  │
      │   NodeC   │
      │     │     │
      └─────┴─────┘

  五、数据流总结

  完整的消息发送-接收流程：

  前端输入消息
    │
    ↓ (React → Wails TypeScript 绑定)
  app.SendDirect(peerID, content)
    │
    ↓
  chat.Service.SendDirect()
    ├─> 派生会话 ID (deriveCID)
    ├─> 获取好友公钥 (懒加载)
    ├─> NaCl Box 加密
    └─> 构造 encWire {cid, sender, ts, nonce, cipher}
    │
    ↓
  nats.Service.Publish("dchat.dm.{cid}.msg", data)
    │
    ↓
  ───────────────── NATS Routes 集群传播 ────────────────
    │
    ↓
  对端节点的 NATS 订阅收到消息
    │
    ↓
  chat.Service.handleEncrypted()
    ├─> 反序列化 encWire
    ├─> 获取解密密钥 (懒加载)
    ├─> NaCl Box 解密
    └─> 构造 DecryptedMessage
    │
    ↓
  触发 OnDecrypted 事件回调
    │
    ↓ (Wails → React)
  runtime.EventsEmit(ctx, "message:decrypted", msg)
    │
    ↓
  前端实时显示消息
