# DChat 功能实现清单

## 📋 功能概述
本文档列出了DChat应用已实现的所有功能，包括底层已实现但上层尚未绑定的功能。

---

## ✅ 已实现并已绑定到上层(Wails API)的功能

### 1. 用户身份管理
| 功能 | 上层API | 描述 |
|------|---------|------|
| 设置用户昵称 | `SetUserInfo(nickname string)` | 修改当前用户的显示昵称 |
| 获取当前用户信息 | `GetUser() (chat.User, error)` | 获取当前用户ID和昵称 |

### 2. NSC密钥与加密功能
| 功能 | 上层API | 描述 |
|------|---------|------|
| 加载NSC密钥 | `LoadNSCKeys(nscSeed string) error` | 从NSC种子派生聊天加密密钥对 |
| 通过NSC公钥添加好友 | `AddFriendNSCKey(uid, nscPubKey string) error` | 仅通过对方NSC公钥即可添加好友，无需提前交换聊天密钥 |
| 直接添加好友聊天公钥 | `AddFriendKey(uid, pubB64 string) error` | 手动添加好友的X25519公钥 |
| 添加群聊对称密钥 | `AddGroupKey(gid, symB64 string) error` | 添加群聊的AES-256对称密钥 |

### 3. 聊天会话管理
| 功能 | 上层API | 描述 |
|------|---------|------|
| 获取会话ID | `GetConversationID(peerID string) (string, error)` | 计算与指定用户的私聊会话ID |
| 加入私聊会话 | `JoinDirect(peerID string) error` | 订阅与指定用户的私聊消息 |
| 加入群聊会话 | `JoinGroup(gid string) error` | 订阅指定群聊的消息 |
| 发送私聊消息 | `SendDirect(peerID, content string) error` | 发送加密的私聊消息，自动保存到本地存储 |
| 发送群聊消息 | `SendGroup(gid, content string) error` | 发送加密的群聊消息，自动保存到本地存储 |

### 4. 本地存储功能
| 功能 | 上层API | 描述 |
|------|---------|------|
| 获取会话历史消息 | `GetMessages(conversationID string, limit int, before *time.Time) ([]*storage.StoredMessage, error)` | 分页获取指定会话的历史消息 |
| 标记消息已读 | `MarkAsRead(conversationID string, before time.Time) error` | 标记指定时间前的会话消息为已读 |
| 获取会话信息 | `GetConversation(conversationID string) (*storage.StoredConversation, error)` | 获取会话的基本信息（最后消息时间、类型等） |
| 搜索消息 | `SearchMessages(query string, limit int) ([]*storage.StoredMessage, error)` | 全文搜索本地存储的消息内容 |

### 5. 网络状态功能
| 功能 | 上层API | 描述 |
|------|---------|------|
| 获取网络状态 | `GetNetworkStatus() (map[string]interface{}, error)` | 获取NATS连接状态、LeafNode运行状态、本地连接地址等 |

### 6. 事件推送（前端接收）
| 事件 | 描述 |
|------|------|
| `message:decrypted` | 收到解密后的消息，包含消息内容、发送者、时间等 |
| `message:error` | 消息处理错误（解密失败、密钥缺失等） |

## ⚠️ 底层已实现但上层未绑定的功能

### 1. NSC密钥派生扩展功能
| 功能 | 底层实现位置 | 描述 |
|------|-------------|------|
| 生成SSL证书 | `nsc_crypto.go:GenerateSSLCertificate()` | 从NSC密钥派生生成自签名SSL证书，可用于HTTPS/TLS服务 |
| 获取SSL密钥对 | `nsc_crypto.go:GetSSLKeyPair()` | 从NSC密钥派生Ed25519类型的SSL密钥对 |
| 从NSC公钥派生SSL公钥 | `nsc_crypto.go:GetSSLPubKeyFromNSCPub()` | 通过对方NSC公钥即可验证其SSL证书有效性 |
| 获取所有派生密钥 | `nsc_crypto.go:GetAllDerivedKeys()` | 获取NSC密钥派生的所有用途密钥（认证、聊天、SSL） |
| 验证NSC种子格式 | `nsc_crypto.go:ValidateNSCSeed()` | 验证NSC种子的格式有效性 |
| 生成群聊对称密钥 | `nsc_crypto.go:GenerateGroupKey()` | 生成安全的AES-256群聊对称密钥 |

### 2. 聊天服务扩展功能
| 功能 | 底层实现位置 | 描述 |
|------|-------------|------|
| 设置用户ID | `service.go:SetUserID()` | 覆盖自动生成的用户ID，用于持久身份或测试场景 |
| 设置本地密钥对 | `service.go:SetKeyPair()` | 直接设置X25519密钥对（不通过NSC派生） |
| 手动保存消息 | `service.go:SaveMessage()` | 手动将消息保存到本地存储（通常自动完成，无需手动调用） |

## 🔍 功能实现说明

### 核心加密体系
- **私聊加密**：X25519密钥交换 + XSalsa20-Poly1305 端到端加密
- **群聊加密**：AES-256-GCM 对称加密
- **密钥派生**：所有密钥均从NSC Ed25519身份密钥派生，无需额外密钥管理
- **消息存储**：所有收发消息自动持久化到本地SQLite数据库，支持全文搜索

### 去中心化特性
- ✅ 无需中心服务器：基于NATS LeafNode + Hub 架构
- ✅ 无需密钥交换：仅通过NSC公钥即可完成端到端加密通信
- ✅ 本地优先：所有消息本地存储，支持离线使用
- ✅ 自动发现：节点自动组成全网状网络，无需手动配置
